## Decisions

Ledger-first: rows 12-18 from the v0.5b Codex prompt were appended and marked `verified 2026-04-29` before v0.5b builder code landed. New rows were added for OSOD audit URL constants, PostgreSQL `pgcrypto`/trigger/revoke/index behavior, `pg_dump`/`pg_restore`, Redis `BGSAVE`, and FHIR AuditEvent/restful-interaction code systems before those constants appeared in code.

DB audit remains source of truth: `osod_audit_events` is modeled as the tamper-resistant append-only log. FHIR `AuditEvent` is emitted by projection helpers and retry queue mechanics, not used as the audit UI read source.

Information Blocking classification is row-level, not gate-level. `ib_actor_classification` defaults to `health-care-provider`, and denied rows require one of the eight named Part 171 exceptions.

Backup/restore is operator-only. No MCP tool can trigger backup, restore, or DR reset; scripts live under `scripts/` and the manual workflow is `workflow_dispatch` only.
