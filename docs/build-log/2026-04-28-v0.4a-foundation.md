## Decisions

Corrected lens model: patient-specific lenses are FHIR `Device`; lab/catalog blueprints are `DeviceDefinition`; materials and coatings are `Substance`; runtime geometry lives in `Device.property[]`. No lab-specific parameter explosion in StructureDefinition slices.

Profile installer: generated v0.4 StructureDefinitions are hydrated with base FHIR snapshots from the running Medplum server before install. This keeps generated profiles valid for Medplum validation without hand-maintaining large base snapshots.

Search parameters: no custom OSOD SearchParameter was minted for v0.4a. Standard Observation `code`, `subject`, `date`, and `focus` are used when supported; `get_lens_fit_history` falls back to subject search plus client-side `Observation.focus` filtering if a backend rejects `focus`.

Verification ledger: prompt-supplied LOINC candidates that did not triangulate were not used in builders. The ledger records verified constants and OSOD-minted URLs used by this slice.
