# v0.4c Ortho-K + Myopia Management Build Log

Date: 2026-04-28

## Slice Summary

Implemented v0.4c as two separate Director modules on the v0.4a contact-lens and ocular-measurement substrate:

- Ortho-K is a device-fitting intervention module. It writes `Device-OrthoKLens` instances, fitting-series `Procedure` resources, child trial `Procedure.partOf` entries, lens-focused fit Observations, version-aware Device.property updates, and inline `AdverseEvent` capture.
- Myopia Management is a longitudinal care-strategy module. It is gated by active `EpisodeOfCare` type `myopia-management`, coordinates interventions through `CarePlan.activity`, writes atropine `MedicationStatement` resources with compounded concentration quantities, records axial-length Observations, and renders treatment comparison, protocol reference, and parent education surfaces.
- Specialty contact-lens UI remains deferred to v0.6. The data path is active through the v0.4a `create_lens_device` substrate and v0.4c golden fixtures.

## Decisions

- Ortho-K and Myopia Management remain separate modules and UI sections. They coordinate only through shared FHIR resources, especially `EpisodeOfCare`, `CarePlan`, `Device`, `DeviceUseStatement`, `Procedure`, and Observations.
- FHIR R4 `CarePlan.activity.reference` cannot target the active Device or MedicationStatement resources needed for treatment comparison, so v0.4c uses `CarePlan.activity.detail` plus OSOD extension `myopia-careplan-activity-intervention` for those links. `CarePlan` remains the canonical treatment-plan resource; no `List` resource was introduced.
- The prompt-requested BCVA over-Rx `Observation.derivedFrom -> DeviceUseStatement` pattern was not implemented literally because FHIR R4 `Observation.derivedFrom` target types do not include `DeviceUseStatement`. v0.4c keeps derivedFrom restricted to the verified target set and uses safe resource references elsewhere.
- Axial length now carries verified OD/OS LOINC codes `64742-0` and `66067-0` while preserving the existing OSOD axial-length code for local query continuity. The initially drafted `76516-4` value was rejected by ledger verification because it is a gestational-age code.
- Specialty CL workflows stay invisible in Director until v0.6, but the data path is verified now through golden fixtures and the MCP write tool.
