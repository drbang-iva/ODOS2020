## Decisions

Ledger numbering: the repo already had v0.5a/v0.5b implementation-specific rows numbered 19-32. To preserve the audit trail, v0.5c prompt rows 19-25 were appended as rows 33-39 with the prompt row number recorded in the verdict column. New v0.5c constants were then verified in rows 40-44 before code referenced them.

No timing constant landed. CMS sources verify that E/M documentation should be completed during the encounter or as soon as possible after, and that scribe/student documentation requires clinician authentication, but v0.5c does not pin a numeric timing bound. Any future timing constraint remains blocked behind a new Mandate 14 row for the applicable NCD/LCD/MAC rule.

FHIR field placement correction: FHIR R4 uses top-level `Provenance.policy`; `agent.policy` belongs to AuditEvent. v0.5c writes attestation/amendment policy URLs to `Provenance.policy`, while the v0.5b audit projection surfaces the same URL through `AuditEvent.agent.policy`.

APPEND relationship: new appended Observations link to the original final Observation with `Observation.derivedFrom`. `Observation.partOf` is not used for Observation-to-Observation succession in R4.

## Mandate 15 Close Audit

Check 1, actor binding: `scribe_write_observation` runs as the scribe identity supplied by the local transcription pipeline and records that identity in `Observation.performer` plus the DB audit row. `clinician_attest_observation`, `amend_observation`, and `append_observation_context` require the clinician id to match `OSOD_SESSION_PRACTITIONER_ID` or the human-provisioned audit actor id. Cross-clinician signing is rejected at the MCP layer with a Mandate 8 / ledger row 20 error.

Check 2, existing-pattern embodiment: the v0.5c state machine is enforced at the MCP layer and emitted into v0.5a Medplum AccessPolicy `writeConstraint`s. Signed Provenance uses the v0.4 Provenance builder and v0.5b restore-integrity signature check. Audit rows use the existing v0.5b `osod_audit_events` schema and FHIR AuditEvent projection; no audit schema change landed.

Check 3, external preconditions: the local-upstream transcription boundary is text-only and Binary references are validated by id, not search. The signing layer remains external and must supply a base64 detached `application/jose` signature blob; v0.5c validates and persists the FHIR signature envelope. CMS E/M timing is resolved as "no numeric constant in v0.5c"; future timing enforcement is blocked pending a specific primary-source rule.
