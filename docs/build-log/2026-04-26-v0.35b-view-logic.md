## Decisions During Implementation

- Kept role context presentation-only. `roles.ts` states that role config is not authorization, MCP tool gating, or clinical-write permission logic.
- Kept Health Concerns out of the UI even though the v0.35a builder supports the category.
- Kept MDM output to Low / Moderate / High threshold counts only. No E/M code suggestions or clinical recommendations are shown.
- Kept Program as the clinical UI label for FHIR EpisodeOfCare.
- Used version-aware PATCH helper paths for edit-in-place updates. No UI delete-and-recreate path was added for diagnosis laterality, code, tier, clinical status, or entered-in-error.
- Preserved the existing Director 3D orbital view; no new timeline or Three.js track was added.
