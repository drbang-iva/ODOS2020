## Decisions

Frames Data population is bulk-file-ingest only in v0.6a. `sync_mode` keeps the forward-compatible enum values, but Frames rows are constrained to `bulk`; `cursor_high_water` remains NULL for Frames sync runs.

The subscription settings surface treats the username as non-secret practice configuration. There is no password field, password secret reference, bearer token, API key, or vendor credential string path.

FHIR catalog identity uses canonical URLs such as `https://osod.dev/catalog/frames/<sku>`. Provenance targets reference FHIR canonical URLs or the ingest Task, not raw SQL row primary keys.

The SQL practice inventory table stores the practice identifier and enforces tenant isolation through RLS. A true relational FK is deferred until OSOD has a first-class practice SQL table; the current Medplum-backed practice identity does not give this migration a stable local FK target without adding a sixth v0.6a table.
