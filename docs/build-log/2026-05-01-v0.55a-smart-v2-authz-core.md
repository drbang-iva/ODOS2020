## Decisions

Ledger first: v0.55 ledger rows 1-15 were verified at session open before builder code landed. Rows 1-11 copy the v0.55a prompt scope; rows 12-15 cover additional code constants for revocation, JWKS/client assertions, authorization-code TTL, and ETag behavior.

Local-only auth server: the SMART routes mount inside the existing local MCP SSE Express process. The server reads `OSOD_SMART_SIGNING_KEY_PATH`, enforces mode `0600`, publishes the local JWKS, and rejects non-local client JWKS URLs. Sandbox client state is mirrored into local Postgres through `osod_smart_clients`; discovery reads the local state layer on each request and updates its ETag when that state changes.

AccessPolicy composition: v0.55a consumes the v0.5a role registry and ProjectMembership parameter model. No v0.5a-substrate-amend was triggered.

Scope review: auto-granted and reduced scopes use the computed effective scope. High-risk reductions enter staged review; practice-admin approval creates a redeemable authorization code and autonomous-agent approval is rejected.
