## Decisions

Questionnaires use canonical `Questionnaire` plus `QuestionnaireResponse`, with summary score Observations derived from the response. No dry-eye score uses `Observation.derivedFrom` to point at a Device.

Meibography writes keep the raw asset as `DocumentReference`; Meiboscore and Arita values live in structured Observations derived from that document.

Treatment series use a parent `Procedure` and child session Procedures linked by `Procedure.partOf`. Session parameters are captured as OSOD extensions for energy, wavelength, and spot count, with UCUM units recorded in the verification ledger.

Ophthalmic medications use `MedicationStatement` with an OSOD supply-type extension for OTC, prescription, and supplement status. The product timeline reads the FHIR resources instead of storing a separate UI-only list.

Adverse events use FHIR `AdverseEvent` now so the dry-eye module stays forward-compatible with the USCDI v7 adverse-event direction.
